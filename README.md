# Backuper

**Backuper** — это утилита на языке Go для автоматизации резервного копирования баз данных. Она поддерживает работу с несколькими хостами, сжатие дампов, гибкие политики хранения (retention policy) и предоставляет REST API для мониторинга.

## Поддерживаемые базы данных

- PostgreSQL
- **ПЛАНИРУЕТСЯ** MySQL
- **ПЛАНИРУЕТСЯ** MongoDB

## Основные возможности

- **Многохостовость:** Поддержка нескольких баз данных с индивидуальными настройками для каждой.
- **Планировщик:** Настройка интервалов запуска бэкапов для каждой БД.
- **Сжатие:** Поддержка сжатия Gzip с настраиваемым уровнем (0-9).
- **Политика хранения:** Автоматическое удаление старых бэкапов при достижении заданного лимита (max_count).
- **REST API:** Проверка работоспособности (health check) и инспекция текущей конфигурации.
- **Логирование:** Структурированные логи (zap) с поддержкой ротации (lumberjack).
- **Контейнеризация:** Готовый Dockerfile и docker-compose.yaml для быстрого развертывания.

## Стек технологий

- **Язык:** Go (Golang)
- **Библиотека бэкапа:** [go-pgdump](https://github.com/JCoupalK/go-pgdump)
- **Конфигурация:** YAML
- **Логирование:** Uber Zap + Lumberjack
- **API Framework:** Gin Gonic

## Установка и запуск

### Предварительные требования

- Go 1.25+
- PostgreSQL (клиентские утилиты `pg_dump` не требуются, используется встроенная библиотека)
- Docker & Docker Compose (опционально)

### Локальный запуск

1. Склонируйте репозиторий.
2. Подготовьте файл конфигурации:
   ```bash
   cp config.example.yaml config.yaml
   ```
3. Установите путь к конфигурации в переменную окружения:
   ```bash
   export config_path=./config.yaml
   ```
4. Запустите приложение:
   ```bash
   go run cmd/api/main.go
   ```

### Запуск через Docker

```bash
make up
```
Это запустит сервис `backuper` и тестовую базу данных `test_db`.

## Конфигурация

Основной файл настроек — `config.yaml`. Пример структуры:

### Секция `logging`
- `level`: Уровень логирования (`info`, `debug`, `error`).
- `rotation_settings`: Настройки ротации файлов логов (`max_size`, `max_count`, `max_age`, `compress`).

### Секция `postgres` (Список хостов)
Для каждой базы данных можно настроить:
- `host`, `port`, `dbname`, `user`, `password`: Данные для подключения.
- `internal`: Интервал между бэкапами в секундах (например, `3600` для ежечасного бэкапа).
- `max_count`: Количество последних копий для хранения (политика удержания).
- `compress`: `true`/`false` для включения Gzip.
- `compress_level`: Уровень сжатия (1-9).
- `timeformat`: Формат времени в имени файла (`timestamp`, `yyyy-mm-dd`, `yyyy-mm-dd-hh:mm:ss`).
- `prefix_filename`: Префикс для имени файла дампа.
- `threads`: Количество потоков для выполнения дампа.

### Секция `storage_config`
- `path`: Базовый путь для хранения бэкапов. Полный путь для конкретной БД формируется как: `{path}/{имя_хоста}/{prefix}_{dbname}_{time}.sql[.gz]`.

## REST API

По умолчанию API доступен на порту `:8080`.

- `GET /health` — Проверка статуса (возвращает `alive`).
- `GET /postgres/backupers` — Список настроенных бэкаперов и их параметры.
- `GET /logging/config` — Текущие настройки логирования.
- `GET /storage/config` — Текущие настройки хранилища.

## Разработка

Для удобства разработки используется `Makefile`:

- `make up` — Сборка и запуск контейнеров.
- `make down` — Остановка контейнеров.
- `make migrate` — Создание тестовой таблицы и наполнение данными (в `test_db`).
- `make restore file=...` — Восстановление базы из дампа.
- `make clean` — Очистка логов и бэкапов.
